# 食譜收藏系統 - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：在日常生活中，經常會在網路上看到想嘗試的食譜，或是想記錄自己的拿手菜，但缺乏一個集中管理的系統，導致需要時常常找不到。因此，需要一個專屬的食譜收藏系統來整理與查詢。
- **目標用戶**：我自己（個人日常使用）。
- **核心價值主張**：提供一個輕量、直覺且方便管理的個人化食譜資料庫，隨時隨地記錄、分類與找尋想做的料理。

## 2. 功能需求

以下為系統的主要功能清單與使用者故事（User Stories）：

1. **收藏與新增食譜**
   - 作為我自己，我希望可以手動新增食譜（包含名稱、食材、步驟、圖片），以便保存自己的私房菜色。
   - 作為我自己，我希望可以透過貼上網址快速匯入外部食譜的連結，以便集中收藏網路上的靈感。

2. **查詢與搜尋食譜**
   - 作為我自己，我希望可以透過關鍵字（如食譜名稱、食材名稱）搜尋食譜，以便在想要料理特定食材時能快速找到對應的食譜。

3. **分類食譜**
   - 作為我自己，我希望可以建立並管理不同的食譜分類（如：家常菜、甜點、湯品、減脂餐），以便有系統地管理與瀏覽所有食譜。

4. **食譜筆記與心得**
   - 作為我自己，我希望可以在每次實作後於食譜中新增個人的筆記或心得（例如：鹽要少放、烤箱溫度調整），以便記錄每次試做後的最佳化調整。

5. **隨機推薦食譜（今晚吃什麼）**
   - 作為我自己，我希望系統能隨機挑選並推薦一道已收藏的食譜，以便解決「不知道今天要吃什麼」的選擇困難症。

## 3. 非功能需求
- **技術限制**：後端採用 Python Flask，前端採用 Jinja2 模板引擎與原生 HTML/CSS/JS，資料庫採用輕量級的 SQLite。
- **效能考量**：作為個人系統，資料量不大，系統頁面載入與搜尋回應時間需極快（小於 1 秒），提供流暢的體驗。
- **安全與備份考量**：由於主要為個人使用，初期不需要複雜的多使用者權限控管，但需要有簡單的方式能夠匯出或備份 SQLite 資料庫檔案，避免珍貴的心血遺失。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 範圍層級 | 包含功能 |
| :--- | :--- | :--- |
| **高** | **Must Have** | 1. 手動新增、編輯、刪除食譜<br>2. 觀看食譜列表與詳細內容<br>3. 食譜分類的建立與指派 |
| **中** | **Should Have** | 1. 關鍵字搜尋功能<br>2. 食譜個人筆記欄位 |
| **低** | **Nice to Have** | 1. 外部網址快速匯入/收藏<br>2. 隨機推薦食譜功能<br>3. 食譜圖片上傳功能 |
